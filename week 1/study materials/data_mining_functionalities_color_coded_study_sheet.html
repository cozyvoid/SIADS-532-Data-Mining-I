Core idea Same data, different task
🔴 OLAP
Main question:
What happened, where, and when?
Best for:
Summarizing and exploring data across dimensions
Typical output:
  • Pivot tables
  • Aggregated summaries
  • Drill-down reports
Key operations:
  • Roll-up
  • Drill-down
  • Slice
  • Dice
  • Pivot
Example:
Sales by region, quarter, and product category
🟠 Association
Main question:
What tends to happen together?
Best for:
Finding co-occurrence patterns and rules
Typical output:
  • Association rules
  • Support
  • Confidence
  • Lift
Remember:
  • Useful for pattern discovery
  • Does not imply causation
Example:
{bread, butter} → {jam}
🟡 Classification
Main question:
Which category does this belong to?
Best for:
Assigning objects to predefined labels
Typical output:
  • Class label
  • Class probability
Learning type:
  • Supervised
  • Needs labeled training data
Example:
Spam vs not spam, approve vs deny
🟢 Prediction
Main question:
What value should we expect?
Best for:
Estimating future or unknown numeric values
Typical output:
  • Numeric estimate
  • Forecast
Learning type:
  • Usually supervised
  • Often regression-style tasks
Example:
House price, next month’s sales, demand forecast
🟦 Clustering
Main question:
What natural groups exist?
Best for:
Finding similar groups without predefined labels
Typical output:
  • Clusters
  • Segments
  • Cluster assignments
Learning type:
  • Unsupervised
  • Useful when labels are unavailable
Example:
Customer segments like budget, loyal, premium
🔵 Ranking
Main question:
Which items should come first?
Best for:
Ordering items by relevance, importance, or priority
Typical output:
  • Ordered list
  • Scores
  • Top-k results
Can be:
  • Supervised or unsupervised
  • Search, recommendation, triage
Example:
Search results, ranked products, risk priority list
🟣 Outlier / Anomaly Detection
Main question:
What does not fit the normal pattern?
Best for:
Finding rare, suspicious, or unusual cases
Typical output:
  • Anomaly flag
  • Anomaly score
  • Ranked unusual cases
Common types:
  • Point anomaly
  • Contextual anomaly
  • Collective anomaly
Example:
Fraudulent transaction, unusual sensor spike, suspicious login
Fast distinctions
CompareKey difference
Classification vs PredictionClassification predicts a category; prediction usually predicts a numeric value.
Classification vs ClusteringClassification uses known labels; clustering finds groups without labels.
OLAP vs Data Mining ModelsOLAP summarizes and explores; mining models discover patterns or make predictions.
Ranking vs ClassificationRanking orders candidates; classification assigns a class.
Clustering vs Outlier DetectionClustering finds common group structure; outlier detection finds rare exceptions.
How to choose
OLAP = summarize
Association = together
Classification = label
Prediction = estimate
Clustering = group
Ranking = order
Outlier = unusual
Memory line:
Summarize, Co-occur, Label, Estimate, Group, Order, Flag Unusual
