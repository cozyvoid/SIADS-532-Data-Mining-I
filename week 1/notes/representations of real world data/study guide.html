Data Representations in Data Mining

Color-coded study sheet for fast review: what each representation is, what it preserves, and when to use it.

🔴 Item Set
Main question:
What items are present?
Best for:
Co-occurrence and frequent combinations
Preserves:
  • Presence / absence
  • Co-occurrence
Usually loses:
  • Order
  • Timing
  • Often frequency
Example:
{milk, bread, eggs}
🟠 Vector / Matrix
Main question:
What features describe the object, and by how much?
Best for:
Most standard machine learning tasks
Preserves:
  • Feature values
  • Counts
  • Weights
Usually loses:
  • Order unless encoded
Example:
<1, 0, 3, 2>
🟡 Sequence
Main question:
In what order did things happen?
Best for:
Behavior paths, next-event prediction, clickstreams
Preserves:
  • Order
  • Repetition
  • Transitions
Usually loses:
  • Rich numeric trends unless added separately
Example:
[search → view item → add to cart → purchase]
🟢 Time Series
Main question:
How does a value change over time?
Best for:
Forecasting, trends, seasonality, anomalies
Preserves:
  • Temporal order
  • Trend
  • Seasonality
Usually loses:
  • Complex relationships unless modeled separately
Example:
Daily temperature or stock prices
🟦 Spatial
Main question:
Where is it?
Best for:
Maps, hotspots, proximity analysis
Preserves:
  • Location
  • Distance
  • Geographic proximity
Usually loses:
  • Time unless added separately
Example:
Crime incidents by latitude/longitude
🔵 Spatiotemporal
Main question:
Where and when did it happen?
Best for:
Trajectories, traffic, movement, outbreaks
Preserves:
  • Location
  • Time
  • Movement patterns
Usually loses:
  • Simplicity; this format can be complex and large
Example:
GPS traces of vehicles
🟣 Graph / Network
Main question:
How are entities connected?
Best for:
Communities, influence, fraud, recommendations
Preserves:
  • Nodes
  • Edges
  • Connectivity
  • Paths
Usually loses:
  • Regular time structure or physical space unless added
Example:
Social network of users
🌸 Stream
Main question:
How do we process data as it arrives?
Best for:
Real-time monitoring and online analysis
Preserves:
  • Arrival order
  • Recent behavior
  • Live updates
Usually loses:
  • Complete history if storage is limited
Example:
Live sensor feed or transaction stream
Fast distinctions
CompareKey difference
Item Set vs VectorItem set focuses on presence; vector can also store counts, weights, and numeric values.
Sequence vs Time SeriesSequence focuses on ordered events; time series focuses on numeric values indexed by time.
Spatial vs GraphSpatial is about physical location; graph is about connections and relationships.
Time Series vs StreamTime series is a data form; stream is a real-time arrival setting.
How to choose the right representation
Item Set = co-occurrence
Vector = features
Sequence = order
Time Series = trends
Spatial = maps
Spatiotemporal = movement
Graph = relationships
Stream = real-time
Memory line:
Presence, Features, Order, Change, Location, Location+Time, Connections, Continuous Arrival
